1.  **プロダクションビルドの失敗:**
    - **現状:** `npm run build` を実行すると、TypeScriptの型エラーによりビルドに失敗する。
    - **原因の可能性:** AIサービスのSDK（ソフトウェア開発キット）が返すデータ（ストリーム）の型と、アプリケ���ション側で定義・期待している型との間に、厳密な不整合が残っている可能性が高いです。特に、非同期で次々とデータが送られてくるストリーム処理の型定義は複雑で、不整合が起きやすい箇所です。
    - **次のステップ:** ビルドエラーメッセージを正確に解読し、原因となっている型定義の不整合を特定・修正する。

2.  **本番環境用の認証設定:**
    - **現状:** ローカル開発では、サービスアカウントのJSONキーファイルを直接参照している (`.env.local`)。
    - **課題:** この方法はCloud Runのような本番環境では利用できず、セキュリティリスクも高いです。
    - **対応方針:** GCPの **Secret Manager** を利用します。サービスアカウントキーのJSONファイルの内容をSecret Managerに保存し、Cloud Runの実行環境に環境変数として安全にマウントする方法に切り替える必要があります。
